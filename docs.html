utext — Documentation & Maintenance

Chat E2EE 1-on-1, backendless (Supabase BaaS), static SPA (Vue 3 + Vite). Semua fitur jalan di free tier Supabase, NO billing, personal use.

1. Arsitektur Singkat

LayerTech
FrontendVue 3 (<script setup>), Vite, Pinia
BackendSupabase: Postgres + Auth + Realtime + Storage
AuthGoogle OAuth
EnkripsiX25519 (libsodium) — keypair RANDOM, tidak diturunkan dari UID
Key backupRaw private key → Google Drive (tombol manual) + cache localStorage
MediaEnkripsi di client → upload bucket media (private)
DeployStatic SPA (Vercel / Netlify)
Alur enkripsi: saat login pertama, generate keypair X25519. Public key → profiles.public_key. Private key → localStorage + Drive backup. Tiap pesan: sharedSecret(publicKey lawan, privateKey kita) → encrypt. Private key TIDAK pernah ke Supabase.

3. Setup / First Run

git clone https://github.com/mocriz/utext.git
cd utext
npm install
cp .env.example .env   # isi SUPABASE_URL, ANON_KEY, GOOGLE_CLIENT_ID
# Jalankan SQL ke Supabase urut: phase-f → g → h → i → j → k
npm run dev           # http://localhost:5173
Wajib: tanpa jalanin SQL phase-f s/d k, fitur berikut GAGAL: reply, edit, read receipt, soft-delete account, delete message, delete conversation, search 1-char.

5. AppShell.vue — State & Handler

State: room{messages,draft,typing,partnerOnline}, activeConv, activePartner, replyingTo, editingMsg, searching/searchQuery/searchResults, sidebarWidth (350–700px), viewerSrc, markedReadFor (Set), seenMsgIds (Set), lastSeen.

Handler:

  • onOpen(c) — load, subscribe, mark receipt SEKALI, setup typing/presence, guard re-subscribe (cegah leak+spam)
  • closeRoom() — unsubscribe semua, revoke blob, clear timers/sets
  • onSend() — kirim / simpan edit
  • onBubbleMenu(m) — lawan: hanya "Hapus untuk saya"; sendiri: + "Hapus untuk semua"
  • onConvMenu(c) — "Hapus untuk saya" / "Hapus untuk semua" (konfirmasi)
  • onCtxSelect(val) — eksekusi + toast
  • startChat(user) — buka/buat dari search (extract UUID)
  • focusComposer() — auto-focus field

6. Supabase SQL — Urutan WAJIB

Supabase → SQL Editor → Run urut:

  1. phase-f — status, deleted_for[], deleted_for_all, deleted_at, RPC mark_read/delivered/set_display_name/soft_delete_account
  2. phase-g — reply_to, edited_at
  3. phase-h — RPC delete_message_for_me, bucket avatars + policy
  4. phase-i — relax search_users 1 char
  5. phase-j — fix delete_message_for_me (NULL-safe)
  6. phase-k — soft_delete_account (final), delete_conversation_for_all
Setelah phase-f: alter publication supabase_realtime add table messages, conversations, conversation_members; biar Realtime jalan.

RPC yang harus ada: create_conversation_with, search_users, mark_read, mark_delivered, set_display_name, soft_delete_account, delete_message_for_me, delete_conversation_for_all.

12. Keamanan & Limitasi

  • E2EE end-to-end: Supabase hanya simpan ciphertext. Admin TIDAK bisa baca isi chat.
  • Private key di localStorage (same-device) + Drive backup. Siapa akses device → bisa baca chat.
  • Free tier: 500MB DB, 1GB storage. Cukup personal.
  • Soft-delete: akun di-flag deleted_at, chat lama tetap bisa dibaca lawan (yang sudah pernah chat). Login Gmail sama → mulai dari nol (public_key di-null).
  • Pastikan RLS aktif di semua table.

13. Account Lifecycle (create / delete / restore)

Create akun

  • Login Google OAuth → Supabase bikin auth.users (id = auth.uid()).
  • ensureIdentity() cek profiles.public_key:
    • NULL → user baru: generate keypair X25519, simpan public_key, generate username random, cache private_key localStorage. Status: new.
    • ada → user lama: pakai public_key, cari private_key localStorage. Ada → ok. Tidak → need_restore.

Hapus akun (reset identitas)

softDeleteAccount() (RPC soft_delete_account + hapus backup):

  • deleted_at = now(), display_name = 'Deleted Account', username = NULL, key_backed_up = false, public_key = NULL.
  • Hapus file backup di Drive (deleteDriveBackup()).
  • Hapus private_key lokal (clearKey()).
  • Lawan: yang SUDAH pernah chat tetap baca chat lama (public key kita sudah di-cache di client mereka). Yang belum pernah buka → chat lama tidak bisa decrypt (edge case). Lawan lihat nama kita = "Deleted Account".

Daftar ulang (email sama)

  • Supabase Google OAuth email sama → auth.uid() SAMA (bukan user baru). Tapi public_key sudah NULL → ensureIdentity() branch "user baru" → generate keypair BARU otomatis → mulai dari nol.
  • Backup Drive sudah dihapus → tidak ke-restore ke profil lama.

Restore vs Mulai Baru

  • Restore dari Drive: restorePrivateKey() → file ada → return key → ok. File hilang → null → status new.
  • Mulai Baru (tombol fallback): startFresh() → ensureIdentity() → generate keypair baru (jalan karena public_key NULL).
  • Stuck prevention: restorePrivateKey() return null (bukan throw) kalau backup hilang → tidak stuck.
Penting: JANGAN pertahankan public_key saat delete — kalau dipertahankan, ensureIdentity balik ke branch "user lama" → loop need_restore (bug). Null-kan agar daftar ulang = user baru otomatis. restoreFromDrive() return privateKey (bukan true).
